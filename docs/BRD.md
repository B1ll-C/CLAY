# CLAY — Business Requirements Document

> Status: Draft — to be completed during V1 planning

## Business Goals

1. Reduce household food waste through expiration date alerts
2. Save users money through cross-store price comparison
3. Reduce duplicate purchases via accurate real-time inventory
4. Reliable offline-first operation (grocery stores have poor signal)

## Target Users

- Home cooks managing pantry and fridge stock
- Families coordinating shared grocery shopping
- Budget-conscious shoppers comparing store prices

## Release Targets

- MVP: Offline single-user app (Phases 1–5, 7) — ~25 dev days
- V1: Backend sync + price comparison (Phases 6, 8) — +20 dev days
- V2: Sharing, push notifications, analytics — +30 dev days
